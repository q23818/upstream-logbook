# ✍️ 内容生产模板库 (Content Templates)

> **职能**：解决“小白营员”写不出、写不好稿件的痛点，提供针对不同平台的标准化 Markdown 骨架。

## 📖 核心逻辑
本模板库的所有内容均对齐《全平台运营与技术文章分发策略》。我们拒绝空洞的宣传，强制要求营员从研发过程（`logs/`）中提取真实的技术决策与避坑经验，将其转化为具有“生命力”的社区内容。

## 📂 模板清单与适用场景

| 模板文件 | 适用平台 | 内容调性 | 核心价值 |
| :--- | :--- | :--- | :--- |
| **[`v2ex-post-template.md`](./v2ex-post-template.md)** | V2EX | 极简、硬核、真诚 | 建立技术信誉，获取高质量反馈 |
| **[`xhs-visual-template.md`](./xhs-visual-template.md)** | 小红书 | 视觉冲击、高情绪、干货 | 捕获大众流量，收藏式传播 |
| **[`zhihu-depth-template.md`](./zhihu-depth-template.md)** | 知乎 | 逻辑严密、行业深度 | 沉淀方法论，获取搜索长尾流量 |
| **[`juejin-tutorial-template.md`](./juejin-tutorial-template.md)** | 掘金 / CSDN | 保姆级、实战导向 | SEO 占位，吸引精准开发者 |

## 🛠️ 如何使用？
1. **选择阵地**：根据你的分发目标（是要 Star、要反馈、还是要流量）选择对应模板。
2. **提取素材**：打开项目的 `logs/` 目录，找到本周最重要的技术决策或 Bug 修复记录。
3. **填空写作**：将素材填入模板中的 `[Meat]` 部分，并保留模板中的结构化指引。
4. **植入追踪**：必须使用 `tools/utm-tracker/utm-gen/` 生成带标签的链接。
5. **核销战果**：发布后 48 小时，通过 `utm-fetch` 审计回流数据并记录至 `report/`。

## 📝 补充内容模板清单

### 1. 技术社区类 (强调硬核与开发者生态)

* **掘金 (Juejin) / CSDN / 思否 (SegmentFault)**：
* **特点**：侧重“系列教程”和“SEO 关键词优化”。
* **模板核心**：代码高亮展示、GitHub Star 引导组件、技术架构图解析。


* **GitHub Discussion / Issue**：
* **特点**：直接在项目主场进行公告和互动。
* **模板核心**：版本更新日志 (Release Notes)、新功能提案说明、Bug 反馈规范。



### 2. 海外分发类 (走向国际化的商业化探索)

* **Product Hunt**：
* **特点**：开源产品“首秀”的最高殿堂，强调产品故事与 Launch 策略。
* **模板核心**：首屏介绍文字 (Tagline)、创始人访谈式背景介绍、Maker 的首条评论回复。


* **Hacker News**：
* **特点**：极度硬核，严禁任何装饰，只看技术深度。
* **模板核心**：简洁的标题、直达源码/Demo 的 URL、极简的技术背景描述。



### 3. 社交与短资讯类 (高频触达与流量捕获)

* **即刻 (Jike) / Twitter (X)**：
* **特点**：碎碎念式的研发进展，维护“有生命力”的人设。
* **模板核心**：#BuildInPublic 标签、进度百分比、一张关键的代码/UI 截图。


* **Bilibili / 视频号 (视频脚本)**：
* **特点**：动态展示项目效果，降低小白的“理解成本”。
* **模板核心**：15秒 Hook (钩子) 脚本、核心功能演示、口播转化引导。



### 4. 商业化与私域类 (沉淀核心用户)

* **微信公众号 (WeChat)**：
* **特点**：长效深度沉淀，适合发布完整的“复盘报告”。
* **模板核心**：排版样式引导、底部二维码/社群引导、往期精选链接。
