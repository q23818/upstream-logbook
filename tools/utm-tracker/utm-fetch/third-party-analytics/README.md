# 方案 C：第三方专业分析集成 (Professional Analytics)

> **定位**：终极方案。适用于拥有项目官网 (Homepage) 或静态文档页 (Docs) 的成熟项目。

## 📖 实现原理
在网页 HTML 中嵌入一段轻量级的统计脚本（如 Umami）。当用户携带 UTM 参数（如 `?utm_source=v2ex`）进入页面时，脚本会自动解析并在后台生成多维度的流量看板。

## 🛠️ 配置步骤
1. **部署统计后台**：使用实验室统一提供的 Umami 实例，或按照 [Umami 官网](https://umami.is/) 自行托管。
2. **嵌入脚本**：将统计代码插入到 `index.html` 或文档模版的 `<head>` 标签中。
3. **保持 UTM 规范**：分发时必须严格执行 `utm_gen.py` 生成的参数格式，否则后台无法归类。

## 📊 数据回收与结账
1. **查看看板**：小白营员登录 Umami 后台，选择对应的项目。
2. **筛选来源**：点击 "Referrers" 或 "UTM Sources" 即可直观对比各平台的引流效果。
3. **同步数据**：将本周的“访客数”与“Star 增长数”进行关联分析，填入 `report/` 周报。

## ✅ 为什么推荐 Umami？
- **隐私保护**：不使用 Cookie，符合海外合规要求。
- **开源透明**：符合实验室“方法论透明化”的价值观。
- **零代码审计**：小白通过图形化界面即可完成数据回收，无需运行 Python 脚本。

> 它不需要我们维护复杂的跳转逻辑，而是利用成熟的开源分析工具（如 Umami 或 Plausible）来处理所有 UTM 数据的可视化。  
> 这种方案的优势在于其**强大的看板功能**，小白营员不需要运行脚本，直接通过浏览器就能看到各个渠道的转化率曲线。
