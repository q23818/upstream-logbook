# 多平台分发指南 (Platform Distribution Guide)

> **说明**：本指南用于指导如何将项目产出转化为全平台影响力，对齐任务书中的内容分发指标。

## 1. 平台分类与内容调性 (Platform Matrix)

| 平台类型 | 核心平台 | 推荐内容类型 | 运营目标 |
| :--- | :--- | :--- | :--- |
| **技术社区** | 掘金、思否、c-s-d-n | 深度技术教程、架构设计、pr 记录 | 获取开发者关注、增加 pr |
| **知识社交** | 知乎、公众号 | 行业趋势分析、开源商业化思考、项目周报 | 建立行业影响力、吸引潜在伙伴 |
| **视频平台** | b 站 (bilibili)、视频号 | demo 演示、技术直播剪辑、营员 v-log | 直观展示产品力 |
| **代码社交** | github (discussions) | 路线图 (roadmap)、决策说明、技术文档 | 沉淀核心社区资产 |

## 2. 内容类型与分发策略 (Content Strategy)

### a. 技术深度类 (12+ 篇指标重点)
- **定义**：解决具体技术问题的干货。
- **分发**：首发掘金 -> 转发知乎 -> 代码片段发 github gist。

### b. 商业/调研类
- **定义**：用户访谈心得、付费意向书背后的思考。
- **分发**：首发公众号/知乎 -> 简版发即刻/朋友圈。

### c. 动态/视频类 (6+ 次指标重点)
- **定义**：产品迭代演示、重大决策宣布。
- **分发**：视频发 b 站 -> 动图/短文发全平台动态。

## 3. 分发检查清单 (Checklist)
- [ ] 标题是否针对不同平台做了微调？
- [ ] 是否已在文末添加项目 github 链接？
- [ ] 是否已在本地 `community-and-outreach/` 目录下存盘草稿？
- [ ] 是否已在 `report/`（周报）中同步数据？
