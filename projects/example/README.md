## 🔄 整体协作工作流 (Standard Workflow)

本项目的运作遵循 **“大脑与肌肉分离”** 原则，分为以下两种执行路径：

### 路径 A：营员主动领任务（从任务池认领）
1. **看池子**：营员在 `daily-planner.md` 的【待领任务池】选取素材。
2. **建工单**：在 **GitHub Issues** 创建 Issue，并反向将 `#IssueID` 填回 Planner。
3. **出稿件**：在 Issue 评论区提交草稿，由负责人审核修改。
4. **做结算**：发布后，先在 **`community-and-outreach/distribution-log.md`** 登记 URL 和数据，最后关闭 Issue。

### 路径 B：负责人指派任务（定向空投）
1. **下军令**：负责人直接在 **GitHub Issues** 创建任务并 `@营员`。
2. **步协同**：营员将该任务同步到 `daily-planner.md` 的【今日执行进度】。
3. **闭环**：后续审核与“成果台账”登记流程同路径 A，确保每一笔开支（精力）都有回报（数据）。
